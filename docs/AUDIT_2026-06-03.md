# Echo Flow — Full-System Audit (2026-06-03)

Triggered by user reports of (a) dictation sometimes coming out "Every Word
Capitalized", (b) the light/dark theme reverting after restart, and (c) the
Folders editor "not working". A complete read-only audit of the dictation
pipeline, the core daemon, and the Flask dashboard was run (three parallel
auditors), findings triaged by confidence/severity, and the high-confidence,
verifiable defects fixed with regression tests.

**Test status:** 833 passing (was 827 at audit start; +6 new regression tests).

---

## Root cause of the three reported symptoms

The current code is largely correct. Direct execution confirmed that:

- Over-capitalized text **is** flattened by the normal pipeline (fast path and
  the LLM-then-finalize path both call `_finalize`).
- The theme toggle **does** persist to disk (`config_writer.set_scalar`
  round-trips `dashboard.theme`), and the dashboard runs **in the daemon
  process** (the PyWebView `window.py` is only a shell pointing at the in-process
  Flask server) — so there is a single config file, written and read by the same
  process.

The reported symptoms are therefore best explained by the **running tray daemon
being a stale build** — it predates the casing-flatten and theme-persistence
work, so in-session toggles work (in-memory) but don't reflect the fixed
on-disk behavior. The user confirmed they had not fully quit/relaunched the
tray app.

> **Action for the user:** fully quit Echo Flow from the tray and relaunch (or
> rebuild the frozen app). The fixes below only take effect in a fresh process.

That said, the audit found **genuine latent bugs** on rarer paths that *would*
reproduce the casing symptom even on current code (e.g. a long dictation that
skips the already-clean fast path, when the model returns a chatbot-style answer
and the hallucination guard trips). Those are now fixed.

## Fixed in this pass (with tests)

| # | Area | Bug | Fix |
|---|------|-----|-----|
| 1 | `cleanup.py` | Hallucination-guard passthrough pasted raw Whisper text without the casing flatten | Run `_finalize` (LLM-free, content-preserving) on passthrough |
| 2 | `cleanup.py` | Total-provider-failure passthrough skipped the flatten | Same, guarded so a polish error can't swallow the user's words |
| 3 | `cleanup.py` | `learned` provider with `fallback_to_ollama: false` (or failed fallback) returned raw text un-finalized | Finalize on both learned-fallback exits |
| 4 | `dashboard/settings_routes.py` | `/settings/*` pages froze the theme at startup | Read `dcfg.get("theme")` live per render |
| 5 | `hotkey.py` | An exception in the activate/deactivate callback killed the pynput listener permanently (silent dictation death) | `_safe()` wrapper logs and swallows |
| 6 | `notes.py` | Semantic backlinks re-queried by non-unique `raw_text`, linking the wrong dictation | Use `retriever.search_with_ids()` |
| 7 | `learn.py` | `_meaningful_casing` rejected curly-apostrophe possessives (`TikTok’s`), dropping the learned casing | Strip all apostrophe glyphs |
| 8 | `dashboard/app.py` | Scratchpad-target `back` field allowed open redirect; flash not URL-encoded | Restrict to same-site relative paths; `quote_plus` the flash |

Regression tests: `tests/test_casing_passthrough.py`, `tests/test_audit_fixes.py`,
and updates to `tests/test_cleanup_fallback_chain.py` /
`tests/test_via_ollama_errors.py` (the failure paths now assert words-preserved +
casing-normalized rather than byte-exact verbatim — consistent with the pipeline's
already-universal casing normalization).

## Deferred (substantiated, but not fixed this pass)

Held back because they are platform-specific and can't be verified without the
live desktop environment, are intentional design choices, or are low-risk
robustness items better done with the user awake to confirm intent.

- **`inject.py` clipboard race / non-ASCII `typewrite`.** On rapid double
  dictation the clipboard-restore thread can race the next paste; `method: type`
  (non-default) silently drops non-ASCII characters. Touching the core paste
  path without desktop testing is risky.
- **`singleton.py` PID file relative path + `os._exit` bypassing `atexit`.**
  On frozen installs the PID file can land in the read-only exe dir; normal quit
  never cleans up the PID file. Affects watchdog only.
- **`learn.py decay_stale` uses SQLite `pow()`.** Available on this machine
  (SQLite 3.50.4) but not on builds compiled without math functions — would make
  decay a silent no-op. Worth a Python-side fallback for older frozen bundles.
- **Flash messages not URL-encoded across ~30 dashboard routes.** Only the
  highest-risk (user-data-bearing) routes were encoded this pass; a sweep of the
  rest is a clean follow-up.
- **`cleanup.py` `system_prompt_override` skips `_finalize`.** Intentional —
  custom transforms own their output formatting (same as PE mode).
- **`config_writer.py`** leaks a `.tmp` file if a round-trip check fails on a
  flow-style collection value; **`db_path`** from config is used without a
  path-traversal guard (local-only, low risk).

## Notes

- `config.yaml` carries the user's personal `theme` toggle as runtime state; it
  is intentionally **not** part of the audit-fix commit.
- No production behavior contract was broken: the "raw passthrough" contract was
  always "never reword the user's text" — casing normalization preserves words
  and only fixes capitalization/punctuation, so it honors that contract.
